Release Information

For more information on the features discussed on this page see our feature page.

Version 0.2

Version 0.2 is the first major release of the software and adds the ability to acquire, store, and display information about electric meter data. Specifically:

  • Added ability to create, edit, and delete groups of meters.
  • Added ability to graph one or more meters and/or groups as a line or bar graph.
  • Added ability to compare one or more meters and/or groups energy usage over a one week interval.
  • Added ability to link to specific charts.
  • Meters and groups now have distinct colors when graphed.
  • Improved mobile support.
  • Added ability to export readings from system.
  • Backend improvements for increased speed, stability, and scalability.
  • Improved meter import process.
  • Other minor improvements and fixes.

Upcoming

You can visit our GitHub repository to see issues and find the ongoing status for each issue. What follows is a summary of our plans for upcoming releases and, for some items, the status of that effort. This was last updated on 8 December 2017.

0.3 Release plans with a release planned for end of January 2018
  • Installs via Docker (done)
  • Improved checking of meter data to exclude potentially bad data. OED also continues on to other meters after any issues encountered. (done)
  • An admin panel to allow someone responsible for the OED system to enter information needed and generally control the way OED functions at their site. (in development and nearing testing)
  • Make the OED website controls and basic wording multilingual. (going into testing)
  • Enhance comparison graphic to allow comparing day, week and four week periods of time. Improve labeling of graphic based on user feedback of initial setup on week periods. (in development)
  • Allow admins to import meter data from a CSV file. This will allow MAMAC logs, data that existed before meters were read by OED, and data from other systems to be imported into OED. (in testing)
  • Provide email warnings of system issues (in development and nearing testing)
0.4 Release plans with a target release sometime around March 2018
  • Generalization of energy types beyond electricity including steam, gas, water, garbage/recycling, etc. (being designed)
  • Allow energy to be represented in other units such as CO2, laundry loads, etc. (in design as part of more energy types)
  • Add the ability for OED to calculate baseline of a meter on a user selectable date range. Groups will automatically combine baselines of the meters involved. You will be allowed to have multiple baselines for a given meter so it can vary with time. All graphics will take baselines into account. (in development)
  • Allow admin to apply a multiplicitive factor to each meter added to a group. This will allow for negative values for production meters and correction of meter values or units.
  • Allow anyone to see what meters and groups are included in a group. Currently only admins can do this.
  • Enhance the admin panel including allowing for addition of meters via a CSV file and input of new meter information from the generalization of meter types
  • Speed up getting meter data for users. This will also allow users to download the raw meter data from a graph rather than just the points shown on the graph.
Actively under development but time frame unknown
  • Ability to read from more meter types including Modbus and BACnet. This is likely to be through third party hardware at this time but we welcome anyone interested in creating another solution to contact us.

You can influence our upcoming features by becoming involved. We also have a list of feature requests that we have yet to decide to implement that can be seen here.
