Preparing an OED Release

In preparing for an OED release, the following steps should be followed:

  1. Releases should correspond to a Milestone for the project.
  2. Announce to the Developer discussion board so everyone can look over the proposed release.
  3. Review the Milestone for this release to make sure all issues are complete or reassigned.
  4. See if there are any package or security issues that need to be addressed.
  5. Do extensive running and testing of OED in a web browser. When possible, test in all major web browsers.
  6. Verify the upgrade from a previous version works using the directions on the latest version of the Admin Upgrading help page. This normally means starting from the previous version on a server. Iif not already there see the Admin Installation help page on doing this where you will need to load in test data, add groups, etc. before upgrading to be sure the database migrates correctly. From the previous version you do the upgrade. After the upgrade make sure everything works correctly including the new features.
  7. Create the GitHub release following the steps in GitHub Managing releases in a repository. Note you can save as a draft for later release.
  8. Add info about release to an OED news items and release pages on the website.
  9. Celebrate.
